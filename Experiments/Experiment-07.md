Experiment 7: Creating React Components using JSX and Props

What is React?
A JavaScript library for building user interfaces using components and a declarative rendering model.

What is a Component?
A reusable, self-contained UI unit (function or class) that returns markup and manages its own logic.

What is JSX?
A syntax extension that looks like HTML within JavaScript and compiles to React.createElement calls.

What are Props?
Short for properties; read-only inputs passed from parent to child components.

What is the purpose of Props?
To configure components and pass data or callbacks from parent to child.

What is a Functional Component?
A React component defined as a plain function that returns JSX and may use hooks.

What is a Parent Component?
A component that renders child components and may pass props to them.

What is a Child Component?
A component rendered by another component (the parent).

How are Props passed?
By adding attributes to a component in JSX, e.g., <Child name="Kamlesh" />.

What is component reusability?
Designing components so they can be used in multiple places with different props.

What is Virtual DOM?
An in-memory lightweight representation of the real DOM that enables efficient updates by diffing and patching.

What is export default?
A module syntax that exports a single default value which can be imported without braces.

What is the App component?
Typically the root component of a React application that composes and routes other components.

Difference between Props and State?
Props are immutable inputs from parents; state is internal, mutable data managed by the component.

Why is React popular?
Because of its component model, efficient rendering via Virtual DOM, large ecosystem, and developer productivity.
