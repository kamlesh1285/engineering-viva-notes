Experiment 10: Java OOP Concepts Implementation

What is OOP?
Object-Oriented Programming, a paradigm organizing code using objects that encapsulate data and behavior.

What are the four pillars of OOP?
Encapsulation, Inheritance, Abstraction, Polymorphism.

What is Encapsulation?
Bundling data and methods together and restricting direct access to internal object state.

What is Inheritance?
A mechanism where one class (subclass) derives properties and behavior from another (superclass).

What is Abstraction?
Hiding complex implementation details and exposing only essential interfaces.

What is Polymorphism?
The ability for objects to be treated as instances of their parent type while executing different behaviors (method overriding or overloading).

What is an Abstract Class?
A class that cannot be instantiated and may contain abstract methods that subclasses must implement.

What is an Abstract Method?
A method declared without implementation that subclasses are required to override.

What is a Constructor?
A special method used to initialize new object instances.

What are Getters and Setters?
Methods that read (get) and update (set) private object fields while controlling access.

What is the extends keyword?
Used in Java to declare that a class inherits from another class.

What is method overriding?
When a subclass provides its own implementation of a superclass method with the same signature.

Why is data hiding important?
It protects internal object state from unintended access or modification, improving robustness and maintainability.

What are the advantages of OOP?
Modularity, reusability, easier maintenance, abstraction, and improved code organization.

What is code reusability?
Using existing code (classes, methods) in new contexts to reduce duplication and effort
