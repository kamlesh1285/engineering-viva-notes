Experiment 11: Creating REST API using Spring Boot

What is Spring Boot?
A Java framework that simplifies building production-grade Spring applications with auto-configuration and embedded servers.

What is a REST API?
An API following REST principles, using HTTP methods and stateless interactions to manage resources.

What is REST architecture?
An architectural style that uses resources, URIs, standard HTTP methods, and statelessness for web services.

What is @RestController?
A Spring annotation that marks a class as a REST controller where methods return response bodies (combines @Controller + @ResponseBody).

What is @GetMapping?
An annotation mapping HTTP GET requests to a handler method.

What is @PostMapping?
An annotation mapping HTTP POST requests to a handler method.

What is @RequestBody?
An annotation that binds the HTTP request body to a method parameter, typically deserialized as an object.

What is JSON?
A text format for data exchange (see above).

What is an endpoint?
A URL/URI exposed by an API that clients can call to perform operations.

What is HTTP?
HyperText Transfer Protocol, the protocol for transferring resources on the web.

What is the default Spring Boot port?
8080.

Difference between GET and POST?
GET retrieves data and should be idempotent and safe; POST submits data to create or change resources and may not be idempotent.

What is auto-configuration in Spring Boot?
Automatic setup of Spring components based on classpath and beans to reduce manual configuration.

Why is Spring Boot used?
To accelerate development with sensible defaults, embedded servers, and easy deployment.

What is Postman?
A tool for testing APIs by sending HTTP requests and inspecting responses.
